🛠 Module 2: Logic Detective Arena

"State နဲ့ Logic အမှားတွေကို ရှာဖွေပြီး React ရဲ့ အလုပ်လုပ်ပုံကို ပိုမိုနက်နဲစွာ နားလည်အောင် လုပ်ဆောင်ပါ။"

Challenge 1: State Mutation

"ကျောင်းသားက ခလုတ်နှိပ်ရင် အမှတ်တိုးအောင် လုပ်ထားပေမယ့် Screen ပေါ်မှာ အမှတ်တွေက လုံးဝတိုးမလာပါဘူး။ ဘာကြောင့်လဲ?"

BUGGY CODE const [points, setPoints] = useState(0);

const handleAddPoint = () => {
  points = points + 10; // ❌
};
အဖြေမှန်ကို ကြည့်မည်
React မှာ State ကို တိုက်ရိုက် (direct assignment) ပြင်လို့မရပါဘူး။ အဲ့ဒီလိုပြင်ရင် UI က Re-render မဖြစ်ပါဘူး။ setPoints ဆိုတဲ့ function ကို သုံးရပါမယ်။
✅ ပြင်ဆင်ရန်: setPoints(points + 10);

Challenge 2: Immediate Execution

"App ပွင့်လာတာနဲ့ ခလုတ်မနှိပ်ရသေးဘဲ Alert ကြီးက တန်းတန်းပေါ်လာနေပါတယ်။ ဘာမှားနေလဲ?"

BUGGY CODE <button onClick={alert('Hi!')}>Click Me</button>
အဖြေမှန်ကို ကြည့်မည်
onClick={alert(...)} လို့ရေးရင် function ကို ချက်ချင်း run လိုက်တာပါ။ ကျနော်တို့က ခလုတ်နှိပ်မှ run စေချင်တာဖြစ်လို့ Function Reference သို့မဟုတ် Arrow Function ကို ပေးရပါမယ်။
✅ ပြင်ဆင်ရန်: onClick={() => alert('Hi!')}

Challenge 3: Missing List Keys

"ကျောင်းသားစာရင်းကို loop ပတ်ပြထားပေမယ့် Console မှာ Warning Error နီနီကြီး တက်နေပါတယ်။"

BUGGY CODE {users.map(u => (
  <li>{u.name}</li>
))}
အဖြေမှန်ကို ကြည့်မည်
React က List ထဲမှာ ဘယ်အရာက ပြောင်းလဲသွားလဲဆိုတာ သိဖို့ အရာတိုင်းမှာ unique ဖြစ်တဲ့ key prop လိုအပ်ပါတယ်။
✅ ပြင်ဆင်ရန်: <li key={u.id}>{u.name}</li>

Challenge 4: State Update Logging

"Count ကို တစ်တိုးလိုက်ပေမယ့် console မှာ ကြည့်ရင် တန်ဖိုးအဟောင်းကြီးပဲ ပေါ်နေပါတယ်။ ဘာကြောင့်လဲ?"

BUGGY CODE setCount(count + 1);
console.log(count); // ပေါ်လာတာက ၁ မဟုတ်ဘဲ ၀ ဖြစ်နေသည်
အဖြေမှန်ကို ကြည့်မည်
React State updates တွေဟာ Asynchronous ဖြစ်ပါတယ်။ setCount ကို ခေါ်လိုက်ပေမယ့် ချက်ချင်း update မဖြစ်သေးပါဘူး။ နောက်တစ်ကြိမ် Re-render ဖြစ်မှသာ တန်ဖိုးအသစ်ကို ရမှာပါ။
မှတ်ချက်: တန်ဖိုးအသစ်ကို ချက်ချင်းသုံးချင်ရင် useEffect ထဲမှာ စောင့်ကြည့်ရပါမယ်။

Challenge 5: Input Value Binding

"Input ထဲမှာ စာရိုက်ပေမယ့် ဘာစာမှ ပေါ်မလာသလို State ထဲကိုလည်း ဒေတာမရောက်ပါဘူး။"

BUGGY CODE <input type="text" onChange={setName} />
အဖြေမှန်ကို ကြည့်မည်
Input change event မှာ event object (e) ပါလာပါတယ်။ အဲ့ဒီထဲကမှ e.target.value ကို ယူပြီး သိမ်းရမှာပါ။
✅ ပြင်ဆင်ရန်: onChange={(e) => setName(e.target.value)}
