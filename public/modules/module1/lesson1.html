ဘာကြောင့် React ကို သင်ရမှာလဲ? 💡

အရင်က Website တွေရေးရင် HTML ဖိုင်ကြီးတစ်ခုထဲမှာ အရှည်ကြီးရေးရပါတယ်။ တစ်ခုခုပြင်ချင်ရင် လိုက်ရှာရတာ ခက်ပါတယ်။ React ကတော့ Website တစ်ခုကို Lego တုံးလေးတွေလို အပိုင်းပိုင်းခွဲပစ်လိုက်ပါတယ်။

React ရဲ့ ထူးခြားချက် (၃) ချက်:

  • Components: Website ကို Header, Footer, Sidebar ဆိုပြီး ခွဲရေးလို့ရတယ်။
  • Speed: Virtual DOM ကြောင့် အရမ်းမြန်သွားတယ်။
  • Job Market: ကမ္ဘာပေါ်မှာ အသုံးအများဆုံး Framework ဖြစ်တယ်။

Tools များ ပြင်ဆင်ခြင်း

၁.၁ - Modern Web Development နှင့် React Setup

၁။ Web Development ရဲ့ ပြောင်းလဲလာပုံ (Evolution)

လွန်ခဲ့တဲ့ ၁၀ နှစ်ကျော်က Website တွေဟာ Static ဖြစ်ပြီး ရိုးရှင်းတဲ့ HTML, CSS နဲ့ JavaScript တို့ကိုပဲ သုံးခဲ့ကြပါတယ်။ ဒါပေမယ့် ခေတ်မီလာတာနဲ့အမျှ Facebook, YouTube လိုမျိုး ရှုပ်ထွေးတဲ့ Web Applications တွေ ပေါ်ပေါက်လာပါတယ်။ ရိုးရိုး JavaScript နဲ့တင် ဒီလို App ကြီးတွေကို ထိန်းသိမ်းဖို့ဆိုတာ မဖြစ်နိုင်တော့ပါဘူး။

အဲ့ဒီမှာ React ပေါ်လာပါတယ်။ React ဟာ UI (User Interface) ကို အပိုင်းပိုင်းခွဲပြီး စနစ်တကျ တည်ဆောက်နိုင်အောင် ကူညီပေးတဲ့ Library တစ်ခုပါ။ သူက Declarative Programming ပုံစံကို သုံးပါတယ်။ ဆိုလိုတာက Browser ထဲက Element တွေကို တစ်ခုချင်းစီ လိုက်ပြင်နေစရာမလိုဘဲ "ငါ့ရဲ့ UI က ဘယ်လိုပုံစံ ဖြစ်ချင်တာလဲ" ဆိုတာကိုပဲ React ကို ပြောပြလိုက်ရုံပါပဲ။ ကျန်တာကို React က နောက်ကွယ်မှာ အလိုအလျောက် စီမံပေးသွားမှာပါ။

၂။ Node.js နှင့် NPM ဆိုတာ ဘာလဲ?

React Project တစ်ခုကို စတင်ဖို့အတွက် Node.js ဆိုတာကို မသိလို့ မရပါဘူး။ JavaScript ဟာ မူလက Browser ထဲမှာပဲ အလုပ်လုပ်တာပါ။ ဒါပေမယ့် Node.js ကြောင့် JavaScript ကို ကျွန်တော်တို့ရဲ့ ကွန်ပျူတာ (Operating System) ပေါ်မှာ တိုက်ရိုက် run လို့ ရသွားပါတယ်။

သတိပြုရန်:

Node.js ဟာ React ရဲ့ အစိတ်အပိုင်း မဟုတ်ပါဘူး။ React ကုဒ်တွေကို build လုပ်ဖို့၊ libraries တွေကို စီမံဖို့အတွက် လိုအပ်တဲ့ ပတ်ဝန်းကျင် (Environment) တစ်ခုသာ ဖြစ်ပါတယ်။

NPM (Node Package Manager) ကတော့ ကမ္ဘာတစ်ဝှမ်းက Developer တွေ ရေးထားတဲ့ code အတုံးအခဲ (Packages) တွေကို လှမ်းယူသုံးလို့ရအောင် လုပ်ပေးတဲ့ စနစ်ပါ။ React ကိုယ်တိုင်ဟာလည်း NPM ထဲက package တစ်ခုပါပဲ။

၃။ Vite: နောက်မျိုးဆက် Build Tool

အရင်က Create React App (CRA) ကို သုံးခဲ့ကြပေမယ့် အခုခေတ်မှာတော့ Vite ဟာ အမြန်ဆုံးနဲ့ အကောင်းဆုံး Tool ဖြစ်လာပါတယ်။ Vite ဟာ ကုဒ်တွေကို Browser မှာ ပြသဖို့အတွက် ပြင်ဆင်တဲ့နေရာမှာ ES Modules တွေကို သုံးတဲ့အတွက် CRA ထက် ၁၀ ဆကနေ အဆ ၁၀၀ ထိ ပိုမြန်ပါတယ်။ Developer တစ်ယောက်အတွက် ကုဒ်ပြင်လိုက်တာနဲ့ Browser မှာ ချက်ချင်းပေါ်လာဖို့ (Hot Module Replacement) ဆိုတာ Vite ရဲ့ အဓိက အားသာချက်ပါ။

Step-by-Step Setup Guide

  1. Node.js ကို စစ်ဆေးပါ: Terminal တွင် node -v ရိုက်ကြည့်ပါ။ မရှိသေးပါက nodejs.org တွင် install လုပ်ပါ။
  2. Project အသစ်ဆောက်ပါ: npm create vite@latest my-react-app -- --template react လို့ ရိုက်ပါ။
  3. Folder ထဲဝင်ပါ: cd my-react-app
  4. Dependencies သွင်းပါ: npm install (ဒါက node_modules ကို ဆောက်ပေးမှာပါ)
  5. App ကို စတင်ပါ: npm run dev လို့ ရိုက်ပြီး ပေးထားတဲ့ link ကို Browser မှာ ဖွင့်ပါ။
