Props: Component တွေဆီ ဒေတာပို့နည်း 📦

Component တစ်ခုကို ရေးထားပြီး ပုံစံတူ ဒေတာမတူအောင် ဖန်တီးချင်ရင် Props ကို သုံးရပါတယ်။ ဥပမာ - ကျောင်းသားကတ် (Student Card) ဆိုတဲ့ Component တစ်ခုကို ရေးထားမယ်။ အဲ့ဒီထဲကို နာမည်တွေ၊ အတန်းတွေကို လှမ်းပို့ပေးတာပါ။

ဥပမာ:

// Parent Component (App.jsx)
<StudentCard name="မောင်မောင်" class="Grade 10" />
<StudentCard name="မမောင်" class="Grade 11" />

// Child Component (StudentCard.jsx)
function StudentCard(props) {
  return <h1>နာမည်: {props.name}</h1>;
}

🎓 Module 1 Final Project

ဒီ Module မှာ သင်ယူခဲ့တဲ့ Component နဲ့ Props ကို သုံးပြီး Student Portfolio Portal တစ်ခုကို React နဲ့ တည်ဆောက်ကြည့်ပါ။ ကျောင်းသား ၃ ယောက်ရဲ့ Card လေးတွေ ပေါ်လာအောင် လုပ်ဆောင်ရမှာ ဖြစ်ပါတယ်။

၁.၃ - Components နှင့် Props: Thinking in React

၁။ UI ကို အပိုင်းအစများအဖြစ် ခွဲခြားမြင်တတ်ခြင်း

React ရဲ့ အဓိက အတွေးအခေါ်ကတော့ "Composition" ပါ။ Website တစ်ခုကို စာမျက်နှာအလိုက် မစဉ်းစားတော့ဘဲ Component လေးတွေ ပေါင်းစပ်ထားတဲ့ ပဟေဠိ (Puzzle) တစ်ခုလို စဉ်းစားရမှာပါ။ Component ဆိုတာ UI အစိတ်အပိုင်းလေးတွေကို JavaScript Function တစ်ခုအနေနဲ့ သီးသန့်ခွဲရေးထားတာ ဖြစ်ပါတယ်။

ဥပမာ - Navbar, Sidebar, Card, Button တို့ဟာ Component တစ်ခုစီပါ။ ဒီလိုခွဲရေးခြင်းအားဖြင့် Button တစ်ခုကို ပြင်ချင်ရင် တစ်နေရာတည်းမှာပဲ ပြင်လိုက်ရုံနဲ့ App တစ်ခုလုံးမှာ လိုက်ပြောင်းသွားမှာ ဖြစ်ပါတယ်။ ဒါကို Reusability (ပြန်လည်အသုံးပြုနိုင်မှု) လို့ ခေါ်ပါတယ်။

၂။ Props: အချက်အလက်များ လက်ဆင့်ကမ်းပေးခြင်း

Component တစ်ခုတည်းကိုပဲ မတူညီတဲ့ အခြေအနေတွေမှာ ပြန်သုံးချင်တဲ့အခါ Props ကို သုံးရပါတယ်။ Props ဆိုတာ Properties ရဲ့ အတိုကောက်ဖြစ်ပြီး Function တစ်ခုဆီကို ပေးပို့လိုက်တဲ့ "arguments" တွေနဲ့ တူပါတယ်။

ဥပမာ - StudentCard ဆိုတဲ့ component ရှိတယ် ဆိုပါစို့။ ကျောင်းသား မောင်မောင်အတွက်ဆိုရင် name="Mg Mg" လို့ ပို့မယ်၊ မမောင်အတွက်ဆိုရင် name="Ma Ma" လို့ ပို့မယ်။ Component က တစ်ခုတည်းပေမယ့် Props တွေပေါ် မူတည်ပြီး UI က မတူညီတဲ့ အချက်အလက်တွေကို ပြသပေးသွားမှာ ဖြစ်ပါတယ်။

"Props are Read-Only." ဆိုတာကို အမြဲမှတ်ထားပါ။ Child Component ဟာ သူ့ဆီရောက်လာတဲ့ Props ကို ပြင်ဆင်ပိုင်ခွင့် မရှိပါဘူး။ ဒါဟာ React ရဲ့ One-way Data Flow စည်းမျဉ်းဖြစ်ပြီး App တွေကို Predictable (မှန်းဆရလွယ်ကူ) စေပါတယ်။

၃။ Functional Components နှင့် Destructuring

ခေတ်သစ် React မှာ JavaScript Function တွေကိုပဲ အဓိက သုံးပါတယ်။ ကုဒ်တွေကို ပိုပြီး ရှင်းလင်းသွားအောင် JavaScript ရဲ့ Destructuring နည်းလမ်းကို သုံးပြီး Props တွေကို လက်ခံလေ့ရှိပါတယ်။

function Student({ name, age }) { ... } လို့ ရေးခြင်းက props.name လို့ တစ်ချိန်လုံး လိုက်ရေးနေစရာ မလိုတော့ဘဲ ကုဒ်ကို ပိုမိုဖတ်ရလွယ်ကူစေပါတယ်။
