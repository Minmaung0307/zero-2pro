Design Systems: Website တစ်ခု၏ ဝိညာဉ် 🎨

🏠 သာဓက: မြို့ပြစီမံကိန်း (City Planning)

မြို့တစ်မြို့ကို စနစ်တကျ တည်ဆောက်တဲ့အခါ လမ်းတွေရဲ့ အကျယ်၊ အိမ်တွေရဲ့ အရောင်၊ လမ်းဘေးက ဆိုင်းဘုတ်တွေရဲ့ ပုံစံဟာ တစ်သမတ်တည်း ရှိနေရပါမယ်။ ဒါမှသာ ကြည့်ရတာ သပ်ရပ်ပြီး နေချင်စဖွယ် ဖြစ်မှာပါ။ အိမ်တစ်အိမ်က ရဲတိုက်ပုံ၊ တစ်အိမ်က တဲပုံ ဆိုရင်တော့ အဲ့ဒီမြို့ဟာ ရှုပ်ထွေးနေပါလိမ့်မယ်။

Website တစ်ခုမှာလည်း Design System ဆိုတာ အဲ့ဒီမြို့ပြစီမံကိန်းပါပဲ။ ခလုတ်တွေရဲ့ ထောင့်ချိုး (Border Radius)၊ စာသားတွေရဲ့ အကွာအဝေး (Spacing)၊ သုံးထားတဲ့ Font အရွယ်အစားတွေဟာ စာမျက်နှာတိုင်းမှာ အတူတူပဲ ဖြစ်နေရပါမယ်။ Tailwind CSS ရဲ့ Utility classes တွေဟာ ဒီ Design System ကို လွယ်လွယ်ကူကူ အကောင်အထည်ဖော်နိုင်အောင် ကူညီပေးတာ ဖြစ်ပါတယ်။

ဘာကြောင့် Consistency (တစ်သမတ်တည်းဖြစ်မှု) က အရေးကြီးတာလဲ?

User တစ်ယောက်ဟာ သင့် Website ကို သုံးတဲ့အခါ "အလေ့အထ" တစ်ခု ဖြစ်သွားပါတယ်။ ဥပမာ- "ခလုတ်အပြာရောင်က အတည်ပြုတာ" လို့ သူက မှတ်သားထားရင်၊ နောက်တစ်မျက်နှာမှာ အတည်ပြုတဲ့ခလုတ်ကို အနီရောင် သွားလုပ်ထားလို့ မရပါဘူး။ ဒီလို တစ်သမတ်တည်း မဖြစ်တာဟာ User ကို မျက်စိလည်စေပြီး သင့် App ကို အသုံးမဝင်ဘူးလို့ ခံစားရစေပါတယ်။

Professional Developer တွေဟာ ကုဒ်မရေးခင် Global Styles တွေကို အရင်သတ်မှတ်ပါတယ်။ Spacing အတွက် gap-4 ကို သုံးရင် နေရာတိုင်းမှာ 4 ကိုပဲ သုံးဖို့ ကြိုးစားပါတယ်။ ဒါမှသာ ကုဒ်တွေဟာ ပိုပြီး "Scalable" (ကြီးထွားနိုင်စွမ်းရှိ) လာမှာ ဖြစ်ပါတယ်။
