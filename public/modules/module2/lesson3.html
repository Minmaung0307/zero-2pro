ဒေတာများကို ပတ်ပြသခြင်း (Lists & Keys) 📜

သင့်ရဲ့ App မှာ သင်ခန်းစာစာရင်း (Lesson List) တွေ၊ ကျောင်းသားစာရင်းတွေကို တစ်ခုချင်းစီ HTML မှာ ထိုင်ရေးမနေပါနဲ့။ JavaScript ရဲ့ .map() ကို သုံးပြီး အလိုအလျောက် ပတ်ထုတ်လို့ ရပါတယ်။

Mapping ဥပမာ:

const lessons = ["Setup", "JSX", "State"];

{lessons.map((lesson) => (
  <li key={lesson}>{lesson}</li>
))}

ဘာလို့ Key လိုတာလဲ? 🔑

React က ဘယ်အရာက အသစ်တိုးလာလဲ၊ ဘယ်အရာက ဖျက်လိုက်တာလဲဆိုတာကို သိဖို့ တစ်ခုချင်းစီမှာ Unique Key လိုအပ်ပါတယ်။ ဒါမှ Website က အလုပ်လုပ်တာ ပိုမြန်မှာပါ။

🚀 Module 2 Capstone Project

အခု သင်ယူခဲ့သမျှ (State, Input, Mapping) ကို စုစည်းပြီး "Student To-Do List" တစ်ခုကို တည်ဆောက်ပါ။ သင်ယူရမယ့် သင်ခန်းစာတွေကို ရိုက်ထည့်မယ်၊ ပြီးသွားရင် ဖျက်မယ် (သို့မဟုတ်) အမှတ်တိုးမယ်။ ဒါဟာ တကယ့် Web App တစ်ခုရဲ့ အခြေခံပါပဲ။

၂.၃ - Lists နှင့် Keys: ဒေတာအစုအဝေးများကို စီမံခြင်း

၁။ JavaScript .map() ကို အသုံးချခြင်း

React မှာ Data တွေဟာ များသောအားဖြင့် Array (စာရင်း) ပုံစံနဲ့ လာတတ်ပါတယ်။ အဲ့ဒီ စာရင်းတွေကို UI အဖြစ် ပြောင်းလဲဖို့အတွက် JavaScript ရဲ့ .map() function ကို အဓိက သုံးပါတယ်။ သူက Array ထဲက data တစ်ခုချင်းစီကို ယူပြီး Component လေးတွေအဖြစ် အလိုအလျောက် ပြောင်းလဲပေးသွားမှာပါ။ ဥပမာ- ကျောင်းသား ၁၀၀ ရှိရင် Card ၁၀၀ ထိုင်ရေးစရာမလိုဘဲ map နဲ့ တစ်ခါတည်း ပတ်ထုတ်လိုက်ရုံပါပဲ။

၂။ Key Prop ဆိုတာ ဘာလဲ? (ဘာလို့ မဖြစ်မနေ လိုတာလဲ?)

List တွေကို ပတ်ပြတဲ့အခါ React က key ဆိုတဲ့ prop တစ်ခု မဖြစ်မနေ တောင်းပါတယ်။ ဒါဟာ Error မဟုတ်ဘဲ Warning ပေးတာပါ။ Key ဆိုတာ List ထဲက item တစ်ခုချင်းစီကို ခွဲခြားနိုင်တဲ့ ID (Identity) တစ်ခုပါ။

ဥပမာ- သင်က List ထဲက အရာတစ်ခုကို ဖျက်လိုက်ရင် ဒါမှမဟုတ် နေရာရွှေ့လိုက်ရင် React က အဲ့ဒီ Key ကိုကြည့်ပြီး ဘယ်အစိတ်အပိုင်းကိုပဲ ပြောင်းလဲရမလဲဆိုတာ ဆုံးဖြတ်ပါတယ်။ Key မပါရင် React က တစ်ခုလုံးကို အသစ်ပြန်ဆွဲရတဲ့အတွက် App နှေးသွားတတ်ပါတယ်။ ဒါကြောင့် အမြဲတမ်း unique ဖြစ်တဲ့ ID တွေကို Key အဖြစ် သုံးပေးရပါမယ်။
