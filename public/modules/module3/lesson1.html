🎯 ရည်ရွယ်ချက်

Website တစ်ခုမှာ PageRefresh မဖြစ်ဘဲ URL တွေအလိုက် Page တွေ ဘယ်လိုပြောင်းလဲသွားလဲဆိုတာ နားလည်ရန်။

✅ ရရှိမည့်အကျိုး

Single Page Application (SPA) တစ်ခုကို Professional ကျကျ လမ်းကြောင်း (Routing) ခွဲတတ်သွားမယ်။

၁။ SPA ဆိုတာ ဘာလဲ?

ရိုးရိုး Website တွေမှာ Link တစ်ခုနှိပ်ရင် Browser က Refresh ဖြစ်ပြီး Page အသစ်ကို တောင်းရပါတယ်။ React ကတော့ SPA (Single Page Application) ဖြစ်ပါတယ်။ Page မပြောင်းဘဲ Component တွေကိုပဲ လဲလှယ်ပြသပေးတာပါ။

၂။ React Router Setup

ဒီစနစ်ကို သုံးဖို့ `react-router-dom` ဆိုတဲ့ Tool ကို Install လုပ်ရပါတယ်။

npm install react-router-dom

အဓိက ပါဝင်ပစ္စည်းများ:

  • BrowserRouter: တစ်အိမ်လုံး (App တစ်ခုလုံး) ကို routing စနစ်အောက် သွင်းတာ။
  • Routes & Route: ဘယ် URL မှာ ဘယ် Component ကို ပြမလဲဆိုတာ သတ်မှတ်တာ။
  • Link: `<a>` tag အစား သုံးရပါတယ်။ ဒါမှ Page Refresh မဖြစ်မှာပါ။
Routing က Website ရဲ့ လမ်းညွှန်ဆိုင်းဘုတ်ပါပဲ...

၃.၁ - SPA နှင့် React Router အခြေခံ

၁။ အဟောင်းနှင့် အသစ် ကွဲပြားချက်

အရင်တုန်းက Website တွေမှာ Link တစ်ခုကို နှိပ်လိုက်ရင် Browser က အစကနေ ပြန်လည် (Reload) ဖြစ်သွားပါတယ်။ ဒါက အချိန်ကုန်စေသလို User Experience ကိုလည်း ထိခိုက်စေပါတယ်။

React ဟာ SPA (Single Page Application) ဖြစ်ပါတယ်။ ဆိုလိုတာက Page အစစ်က တစ်ခုတည်းရှိပြီး အတွင်းက အစိတ်အပိုင်း (Components) တွေကိုပဲ လဲလှယ်ပေးသွားတာပါ။ ဒါကြောင့် User က စာမျက်နှာတွေ ကူးပြောင်းနေပေမယ့် Browser က Reload မဖြစ်တော့ဘဲ Application တစ်ခု သုံးနေရသလို အရမ်းကို ချောမွေ့နေမှာပါ။

၂။ React Router DOM ၏ အရေးကြီး အစိတ်အပိုင်းများ

  • BrowserRouter: တစ်အိမ်လုံးကို Routing စနစ်အောက် သွင်းလိုက်တာပါ။ App ရဲ့ အပေါ်ဆုံးမှာ တစ်ခါပဲ ပတ်ရပါတယ်။
  • Routes & Route: "ဘယ် URL လမ်းကြောင်းမှာ ဘယ် Component ကို ပြပါ" ဆိုပြီး သတ်မှတ်ပေးတာပါ။
  • Link: <a> tag အစား သုံးရပါတယ်။ သူက Page reload မဖြစ်အောင် တားပေးတဲ့ အထူးခလုတ်ပါ။
