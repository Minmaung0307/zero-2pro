Git & GitHub အခြေခံမှ အချောသတ်အထိ 🛠️

၁။ Git ကို စက်ထဲထည့်သွင်းခြင်း (Installation)

Git ဆိုတာ software တစ်ခုပါ။ သင့်ကွန်ပျူတာမှာ အရင်ဆုံး ရှိနေဖို့ လိုပါတယ်။

Windows အတွက်

Official Website ကနေ Download ဆွဲပြီး Install လုပ်ပါ။

Download Git

Mac အတွက်

Terminal တွင် အောက်ပါကုဒ်ကို ရိုက်ထည့်ပါ။

brew install git

အောင်မြင်ကြောင်း စစ်ဆေးရန်:

Terminal တွင် git --version ဟု ရိုက်ပါ။ ဗားရှင်းနံပါတ် ပေါ်လာလျှင် အိုကေပါသည်။

၂။ Git ကို ကိုယ်ရေးမှတ်တမ်း ပေးခြင်း (Configuration)

Git ဟာ ဓာတ်ပုံရိုက်ပေးတဲ့ ကင်မရာနဲ့ တူပါတယ်။ အဲ့ဒီဓာတ်ပုံကို ဘယ်သူရိုက်တာလဲဆိုတာ သိဖို့ သင့်နာမည်နဲ့ Email ကို Git ထဲမှာ တစ်ကြိမ်တည်း သတ်မှတ်ပေးရပါမယ်။ (ဒါကို Git Account ဆောက်တယ်လို့လည်း ခေါ်ကြပါတယ်။)

git config --global user.name "သင့်နာမည်"

git config --global user.email "သင့်အီးမေးလ်"

၃။ GitHub Account တည်ဆောက်ခြင်း

GitHub.com ကို သွားပါ။ Sign up ကို နှိပ်ပြီး သင့်ရဲ့ Gmail နဲ့ အကောင့်ဖွင့်ပါ။

GitHub ဆိုတာ သင့်ရဲ့ ကုဒ်တွေကို သိမ်းမယ့် "အွန်လိုင်းသိုလှောင်ရုံ" ပါ။ အကောင့်ဖွင့်ပြီးရင် New Repository ကို နှိပ်ပြီး ပရောဂျက် နာမည်ပေးကာ "သိုလှောင်ရုံ" တစ်ခု စတင်ဆောက်လုပ်ပါ။

"GitHub is where the world builds software."

၄။ Local နှင့် GitHub ချိတ်ဆက်ပုံ (The Handshake)

သင့်စက်ထဲက ကုဒ်တွေ GitHub ဆီ ရောက်ဖို့ လမ်းကြောင်း ချိတ်ပေးရပါမယ်။ GitHub မှာ Repository ဆောက်ပြီးတာနဲ့ ရလာတဲ့ Link ကို ယူပြီး Terminal မှာ အောက်ပါအတိုင်း ရိုက်ပါ။

# ၁။ Git ကို စတင်အသက်သွင်းမယ်

git init

# ၂။ GitHub Link နဲ့ ချိတ်ဆက်မယ်

git remote add origin https://github.com/username/repo-name.git

# ၃။ ကုဒ်တွေကို ပို့လိုက်မယ်

git push -u origin main

Authentication (Login) ပြဿနာ:

ပထမဆုံးအကြိမ် Push လုပ်တဲ့အခါ GitHub က Password တောင်းပါလိမ့်မယ်။ အခုနောက်ပိုင်း GitHub က Password အစစ်ကို လက်မခံတော့ဘဲ Personal Access Token (PAT) ကိုပဲ သုံးခိုင်းပါတယ်။
(Settings > Developer Settings > Personal Access Tokens မှာ သွားထုတ်ရပါမယ်။)

၅။ Bug Hunter: အဖြစ်များသော အမှားများနှင့် ပြုပြင်ပုံ 🛠️

"fatal: not a git repository"

အကြောင်းရင်း: git init မလုပ်ရသေးဘဲ တခြား command တွေ ရိုက်မိလို့ပါ။

✅ ဖြေရှင်းနည်း: git init အရင်ရိုက်ပါ။

"remote origin already exists"

အကြောင်းရင်း: Link အဟောင်းကြီး တစ်ခု ရှိနေလို့ပါ။

✅ ဖြေရှင်းနည်း: git remote remove origin အရင်ရိုက်ပါ။

"Permission denied (publickey)"

အကြောင်းရင်း: SSH Key မရှိတာ ဒါမှမဟုတ် မချိတ်ရသေးတာပါ။

✅ ဖြေရှင်းနည်း: HTTPS link ကိုပဲ သုံးပါ (သို့မဟုတ်) SSH Key အသစ်ထုတ်ပါ။

"failed to push some refs"

အကြောင်းရင်း: GitHub ပေါ်မှာ သင့်ဆီမှာ မရှိတဲ့ဖိုင်တွေ ရှိနေလို့ပါ။

✅ ဖြေရှင်းနည်း: git pull origin main အရင်လုပ်ပါ။

📺 SSH Key ချိတ်ဆက်ပုံ ဗွီဒီယို
