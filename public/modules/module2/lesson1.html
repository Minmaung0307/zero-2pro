useState: App ကို အသက်သွင်းခြင်း ⚡

ရည်ရွယ်ချက်

User တစ်ခုခုလုပ်လိုက်တိုင်း (ဥပမာ-ခလုတ်နှိပ်ခြင်း) UI က refresh မဖြစ်ဘဲ ဘယ်လိုလိုက်ပြောင်းလဲသွားလဲဆိုတာ နားလည်ရန်။

ရရှိမည့်အကျိုး

React ရဲ့ အသက်သွေးကြောဖြစ်တဲ့ State Management ကို အခြေခံကျကျ တတ်မြောက်သွားမယ်။

၁။ State ဆိုတာ ဘာလဲ?

ရိုးရိုး HTML/JS မှာဆိုရင် စာသားတစ်ခုပြောင်းဖို့ ID တွေနဲ့ လှမ်းခေါ်ပြီး ရှုပ်ရှုပ်ထွေးထွေး လုပ်ရပါတယ်။ React မှာတော့ Component တိုင်းမှာ "Memory" (မှတ်ဉာဏ်) တစ်ခု ရှိပါတယ်။ အဲ့ဒါကို State လို့ ခေါ်ပါတယ်။

ဥပမာ - ကျောင်းသားတစ်ယောက်ရဲ့ ရမှတ် (Point) ဟာ State တစ်ခုပါ။ ခလုတ်နှိပ်လိုက်လို့ အမှတ်တိုးသွားတာနဲ့ UI မှာရှိတဲ့ အမှတ်က အလိုအလျောက် လိုက်ပြောင်းသွားပါလိမ့်မယ်။

၂။ useState အသုံးပြုပုံ

useState ကို သုံးချင်ရင် React ဆီကနေ အရင်ခေါ်ယူရပါတယ်။

const [count, setCount] = useState(0);
  • count: လက်ရှိတန်ဖိုး (ဥပမာ- ၀)
  • setCount: တန်ဖိုးကို ပြောင်းလဲပေးမယ့် Function
  • useState(0): စစချင်း တန်ဖိုးကို ၀ လို့ သတ်မှတ်တာပါ။

The Rules of Hooks

useState သုံးတဲ့အခါ လိုက်နာရမယ့် စည်းမျဉ်းများ

၂.၁ - useState: Component ၏ မှတ်ဉာဏ်ကို နားလည်ခြင်း

၁။ Variable နှင့် State ၏ ကွဲပြားခြားနားချက်

JavaScript မှာ ကျွန်တော်တို့ variable တွေ သုံးဖူးကြပါတယ်။ ဒါပေမယ့် React မှာ ရိုးရိုး variable တစ်ခုရဲ့ တန်ဖိုးကို ပြောင်းလိုက်ရုံနဲ့ UI (User Interface) က လိုက်မပြောင်းပါဘူး။ ဘာကြောင့်လဲဆိုတော့ React က အဲ့ဒီ variable ပြောင်းသွားတာကို မသိလို့ပါ။

State ဆိုတာကတော့ React က အမြဲတမ်း စောင့်ကြည့်နေတဲ့ "အထူးမှတ်ဉာဏ်" တစ်ခုပါ။ State တန်ဖိုးတစ်ခု ပြောင်းလဲသွားတာနဲ့ React က "ဪ... UI ကို ပြန်ဆွဲဖို့ လိုပြီ" ဆိုပြီး ချက်ချင်းသိပါတယ်။ ဒါကို Re-render ဖြစ်တယ်လို့ ခေါ်ပါတယ်။ ဒါကြောင့် React မှာ အပြန်အလှန် သက်ရောက်မှုရှိတဲ့ (Interactive) App တွေလုပ်ဖို့ State ဟာ အခြေခံအကျဆုံး ဖြစ်ပါတယ်။

၂။ useState Hook ၏ တည်ဆောက်ပုံ (Syntax)

useState ဟာ React က ပေးထားတဲ့ function တစ်ခုဖြစ်ပြီး array တစ်ခုကို return ပြန်ပေးပါတယ်။ အဲ့ဒီ array ထဲမှာ အစိတ်အပိုင်း (၂) ခု ပါဝင်ပါတယ်-

const [state, setState] = useState(initialValue);
  • state: လက်ရှိ သိမ်းဆည်းထားတဲ့ တန်ဖိုး (ဥပမာ- ရမှတ် ၀)
  • setState: အဲ့ဒီတန်ဖိုးကို အသစ်ပြင်ဖို့ သုံးရမယ့် function (ဥပမာ- ရမှတ်ကို ၁၀ ဖြစ်အောင် ပြင်တာ)
  • initialValue: App စပွင့်ချင်းမှာ ရှိစေချင်တဲ့ အစောဆုံး တန်ဖိုး။

၃။ အရေးကြီးသော စည်းမျဉ်း- Immutable State

React မှာ State ကို တိုက်ရိုက်သွားပြင်လို့ မရပါဘူး (ဥပမာ- count = 5 လို့ မရေးရပါ)။ ဘာလို့လဲဆိုတော့ အဲ့ဒီလို ပြင်လိုက်ရင် React က အပြောင်းအလဲကို မသိလိုက်ဘဲ UI ကို update မလုပ်ပေးလို့ပါ။ အမြဲတမ်း setCount(5) ဆိုတဲ့ setter function ကိုပဲ သုံးရပါမယ်။ ဒါကို Immutability လို့ ခေါ်ပြီး ကုဒ်တွေကို ပိုပြီး စိတ်ချရစေပါတယ်။
