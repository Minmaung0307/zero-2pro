Cloud Entry: Firebase ဖြင့် Login စနစ် တည်ဆောက်ခြင်း 🔥

🎯 ရည်ရွယ်ချက်

Backend Code တွေအများကြီး ရေးစရာမလိုဘဲ စိတ်ချရတဲ့ User Login/Register စနစ်ကို ထည့်သွင်းတတ်ရန်။

✅ ရရှိမည့်အကျိုး

ကျောင်းသားတစ်ယောက်ချင်းစီမှာ ကိုယ်ပိုင်အကောင့်ရှိလာပြီး သူတို့ရဲ့ အမှတ်တွေကို Cloud ပေါ်မှာ သိမ်းဆည်းနိုင်တော့မယ်။

၁။ Firebase ဆိုတာ ဘာလဲ?

Firebase ဆိုတာ Google က ထုတ်ထားတဲ့ BaaS (Backend as a Service) တစ်ခုပါ။ ကျွန်တော်တို့က Frontend (React) ကိုပဲ အာရုံစိုက်ပြီး Database နဲ့ Login စနစ်တွေကို Firebase က အကုန်လုပ်ပေးပါတယ်။

၂။ Authentication (လူသိအောင်လုပ်ခြင်း)

Email နဲ့ Password သုံးပြီး Login ဝင်ဖို့အတွက် `firebase/auth` ကို သုံးရပါတယ်။

createUserWithEmailAndPassword(auth, email, password);
signInWithEmailAndPassword(auth, email, password);

၆.၁ - Firebase Authentication: လုံခြုံသော Login စနစ်

၁။ Backend as a Service (BaaS)

Frontend Developer တစ်ယောက်အတွက် ကိုယ်ပိုင် Server ရေးဖို့၊ Database တွေ စီမံဖို့ဆိုတာ အချိန်အများကြီး ပေးရပါတယ်။ Google ရဲ့ Firebase ကတော့ အဲ့ဒီအလုပ်တွေကို အကုန်လုပ်ပေးပါတယ်။ ကျွန်တော်တို့က ကုဒ်အနည်းငယ်နဲ့တင် အဆင့်မြင့် Login စနစ်တွေ၊ Database တွေကို အသုံးပြုနိုင်ပါတယ်။

၂။ Social Login ၏ အားသာချက်

Gmail နဲ့ Login ဝင်တဲ့စနစ် (Google Auth) ဟာ User တွေအတွက်ရော Developer တွေအတွက်ပါ အကျိုးရှိပါတယ်။ User တွေက Password တွေ အလွတ်မှတ်နေစရာ မလိုတော့သလို၊ Developer တွေအနေနဲ့လည်း Password တွေကို လုံခြုံအောင် သိမ်းဆည်းရမယ့် တာဝန်ကြီးကနေ လွတ်ကင်းသွားပါတယ်။ Firebase ဟာ ဒီလုပ်ငန်းစဉ် တစ်ခုလုံးကို လုံခြုံစိတ်ချရတဲ့ နည်းပညာတွေနဲ့ စီမံပေးထားပါတယ်။
