BUGGY CODE apiKey: "AIzaSyB-Your-Very-Secret-Key-123" // ❌ Hardcoded
အဖြေမှန်ကို ကြည့်မည်
API Key တွေကို ကုဒ်ထဲမှာ တိုက်ရိုက်ရေးပြီး GitHub တင်လိုက်ရင် လူတိုင်း မြင်သွားနိုင်ပါတယ်။ ဒါကြောင့် Environment Variables (.env) ဖိုင်ထဲမှာ သိမ်းရပါမယ်။
✅ ပြင်ဆင်ရန်: apiKey: import.meta.env.VITE_FIREBASE_API_KEY

BUGGY CODE const auth = getAuth(); // ❌
အဖြေမှန်ကို ကြည့်မည်
getAuth() ကို သုံးဖို့အတွက် အရင်ဆုံး initialize လုပ်ထားတဲ့ app instance ကို ပေးပို့ဖို့ လိုအပ်ပါတယ်။
✅ ပြင်ဆင်ရန်: const auth = getAuth(app);

Challenge 4: Race Condition (The Ghost Data)

"Data သိမ်းလိုက်ပေမယ့် ချက်ချင်း Page Refresh လုပ်ရင် Data အသစ်က မပေါ်လာပါဘူး။ ဘာကြောင့်လဲ?"

BUGGY CODE db.collection('users').add(user);
window.location.reload(); // ❌
အဖြေမှန်ကို ကြည့်မည်
Database ထဲမှာ Data သိမ်းတဲ့အလုပ် မပြီးခင်မှာပဲ Refresh အရင် ဖြစ်သွားလို့ပါ။ ဒါကို Race Condition လို့ ခေါ်ပါတယ်။
✅ ပြင်ဆင်ရန်: await သုံးပြီးမှသာ နောက်တစ်ဆင့် (Refresh/Redirect) ကို သွားရပါမယ်။
