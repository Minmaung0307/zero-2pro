useEffect: Component ရဲ့ ဘဝဖြစ်စဉ်ကို ထိန်းချုပ်ခြင်း 🔄

🎯 ရည်ရွယ်ချက်

App စတက်လာတဲ့အချိန် (သို့မဟုတ်) Data တစ်ခုခုပြောင်းသွားတဲ့အချိန်မှာ အလုပ်တစ်ခုခုကို အလိုအလျောက် ခိုင်းစေတတ်ရန်။

✅ ရရှိမည့်အကျိုး

API ခေါ်ယူခြင်း၊ Timer ပေးခြင်းနှင့် အခြား ပြင်ပချိတ်ဆက်မှု (Side Effects) များကို စနစ်တကျ ကိုင်တွယ်တတ်လာမည်။

၁။ useEffect ဆိုတာ ဘာလဲ?

React မှာ UI ဆွဲတာအပြင် အခြားအလုပ်တွေ (ဥပမာ- Data သွားယူတာ၊ ခေါင်းစဉ်ပြောင်းတာ) ကို Side Effects လို့ ခေါ်ပါတယ်။ `useEffect` က အဲ့ဒီအလုပ်တွေကို ဘယ်အချိန်မှာ လုပ်ရမလဲဆိုတာ ဆုံးဖြတ်ပေးပါတယ်။

၂။ Dependency Array (အရေးကြီးဆုံး အစိတ်အပိုင်း)

useEffect ရဲ့ ဒုတိယ parameter မှာပါတဲ့ `[]` ကို Dependency Array လို့ ခေါ်ပါတယ်။

  • useEffect(fn, []): Component စတက်လာတဲ့ (Mount) အချိန်မှာ တစ်ကြိမ်တည်း ပဲ အလုပ်လုပ်မယ်။ (API ခေါ်ဖို့ အကောင်းဆုံးပဲ)။
  • useEffect(fn, [count]): `count` ဆိုတဲ့ state ပြောင်းတိုင်း ပြန်ပြန်ပြီး အလုပ်လုပ်မယ်။
useEffect(() => {
  console.log("App စတင် ပွင့်လာပါပြီ");
}, []);
useEffect မပါရင် API ခေါ်လို့ မရပါဘူး...

၄.၁ - useEffect: Component ၏ ဘဝဖြစ်စဉ်ကို ထိန်းချုပ်ခြင်း

၁။ Side Effects ဆိုတာ ဘာလဲ?

React Component တစ်ခုရဲ့ အဓိက အလုပ်က UI (HTML) ကို ထုတ်ပေးဖို့ပါ။ ဒါပေမဲ့ UI ဆွဲတာအပြင် အခြားအလုပ်တွေ (ဥပမာ- API ဆီက ဒေတာသွားယူတာ၊ Timer ပေးတာ၊ ခေါင်းစဉ် (Title) ကို ပြောင်းလဲတာ) တွေကို လုပ်ဆောင်ဖို့ လိုအပ်လာပါတယ်။ ဒါတွေကို React မှာ Side Effects လို့ ခေါ်ပါတယ်။

useEffect Hook ဟာ ဒီလို Side Effect တွေကို စနစ်တကျ ကိုင်တွယ်ဖို့ ဖန်တီးထားတာပါ။ သူက Component တစ်ခု Browser ပေါ်မှာ ပေါ်လာတဲ့အချိန် (Mount)၊ ပြောင်းလဲသွားတဲ့အချိန် (Update) နဲ့ ပျောက်ကွယ်သွားတဲ့အချိန် (Unmount) တွေမှာ ဘာလုပ်ရမလဲဆိုတာကို ခိုင်းစေနိုင်တဲ့ "အာရုံခံကိရိယာ" လိုပါပဲ။

၂။ Dependency Array ၏ လျှို့ဝှက်ချက်

useEffect မှာ အရေးကြီးဆုံးက ဒုတိယ Parameter ဖြစ်တဲ့ Array `[]` ပါပဲ။ ဒါကို Dependency Array လို့ ခေါ်ပါတယ်။

  • useEffect(fn, []) : Empty Array ထည့်ထားရင် Component စပွင့်တဲ့အချိန်မှာ "တစ်ကြိမ်တည်း" ပဲ အလုပ်လုပ်ပါမယ်။ API ခေါ်ဖို့ အကောင်းဆုံး အချိန်ပါ။
  • useEffect(fn, [count]) : Array ထဲမှာ state တစ်ခုခု ထည့်ထားရင် အဲ့ဒီ state ပြောင်းတိုင်း effect က ပြန်အလုပ်လုပ်ပါမယ်။
  • useEffect(fn) : Array မပါရင် component render ဖြစ်တိုင်း အမြဲ အလုပ်လုပ်နေမှာဖြစ်လို့ Infinite Loop ဖြစ်တတ်တဲ့အတွက် သတိထားရပါမယ်။

၃။ Cleanup Function (သန့်ရှင်းရေးလုပ်ခြင်း)

တချို့ Side Effect တွေ (ဥပမာ- setInterval သို့မဟုတ် Event Listener များ) ဟာ Component ပျောက်သွားရင်တောင် Browser နောက်ကွယ်မှာ ဆက်အလုပ်လုပ်နေတတ်ပါတယ်။ ဒါဟာ Memory တွေကို ဝါးမျိုစေပါတယ်။ အဲ့ဒါကို တားဆီးဖို့ useEffect ထဲကနေ return () => { ... } ဆိုပြီး function တစ်ခု ပြန်ပေးရပါတယ်။ ဒါကို Cleanup Function လို့ ခေါ်ပြီး Component ပိတ်သွားတာနဲ့ အနောက်က ပစ္စည်းတွေကို အကုန်သိမ်းကျုံးရှင်းလင်းပေးသွားမှာ ဖြစ်ပါတယ်။
