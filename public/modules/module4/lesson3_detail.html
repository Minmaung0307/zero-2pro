Dashboard သို့ ပြန်သွားရန်

UX: User စိတ်မညစ်အောင် စီမံခြင်း ⏳

🏠 သာဓက: စားသောက်ဆိုင်က မှာယူမှု အခြေအနေ

အင်တာနက်ကနေ ဒေတာယူတာဟာ စားသောက်ဆိုင်မှာ ဟင်းမှာတာနဲ့ တူပါတယ်။ မှာပြီးတာနဲ့ ချက်ချင်း ရောက်မလာပါဘူး။

၁။ **Loading:** စားပွဲထိုးက "ခဏစောင့်ပါ၊ ချက်နေပါတယ်" လို့ ပြောတာဟာ Loading Spinner လေး ပြထားတာနဲ့ တူပါတယ်။

၂။ **Error:** "ဒီနေ့ ဒီဟင်း ကုန်သွားပါပြီ" လို့ ပြောတာဟာ Error Message ပြတာပါ။

ဘာမှ မပြဘဲ ထားလိုက်ရင် User က App ပျက်သွားပြီအထင်နဲ့ ထွက်သွားပါလိမ့်မယ်။ ဒါကြောင့် ဒီအခြေအနေတွေကို သေချာ ကိုင်တွယ်ရတာ ဖြစ်ပါတယ်။
