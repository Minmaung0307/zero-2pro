Git & GitHub: ကုဒ်များကို စနစ်တကျ ထိန်းသိမ်းခြင်း 🛠️

၁။ Git နှင့် GitHub ဆိုတာ ဘာလဲ?

Developer အများစုဟာ ကုဒ်တွေရေးရင်း မှားသွားတဲ့အခါ အရင်က အလုပ်လုပ်တဲ့ အခြေအနေကို ပြန်သွားချင်ကြပါတယ်။ Git ဆိုတာ သင့်ကွန်ပျူတာထဲမှာရှိတဲ့ ကုဒ်တွေကို အချိန်နဲ့အမျှ "ဓာတ်ပုံရိုက်ပြီး သိမ်းပေးထားတဲ့ အချိန်စက် (Time Machine)" လိုပါပဲ။

GitHub ကတော့ အဲ့ဒီ သိမ်းဆည်းထားတဲ့ ကုဒ်တွေကို အင်တာနက် (Cloud) ပေါ်မှာ တခြားသူတွေနဲ့ မျှဝေလို့ရအောင်၊ အဖွဲ့လိုက် အလုပ်လုပ်လို့ရအောင် လုပ်ပေးတဲ့ "Social Media for Code" လို နေရာမျိုး ဖြစ်ပါတယ်။

အဓိက ကွဲပြားချက်:

  • Git: Tool တစ်ခုဖြစ်သည် (သင့်စက်ထဲတွင်ရှိသည်)။
  • GitHub: Website တစ်ခုဖြစ်သည် (အွန်လိုင်းပေါ်တွင်ရှိသည်)။

၂။ မဖြစ်မနေ သိထားရမည့် Git Commands များ

ကုဒ်တွေကို သိမ်းဆည်းဖို့အတွက် Terminal မှာ ဒီအဆင့်တွေကို အစဉ်လိုက် လုပ်ရပါတယ်။

git init

ပရောဂျက်အသစ်တစ်ခုမှာ Git ကို စတင်အသုံးပြုရန် (အုတ်မြစ်ချခြင်း)။

git add .

ပြင်ဆင်ထားသမျှ ကုဒ်အားလုံးကို သိမ်းဆည်းရန် အသင့်ပြင်ခြင်း (ပစ္စည်းစုခြင်း)။

git commit -m "first commit"

လက်ရှိအခြေအနေကို မှတ်တမ်းအဖြစ် အတည်ပြုသိမ်းဆည်းခြင်း (ဓာတ်ပုံရိုက်လိုက်ခြင်း)။

git push origin main

သင့်စက်ထဲက ကုဒ်တွေကို GitHub ပေါ်သို့ တင်ပို့လိုက်ခြင်း (Upload လုပ်ခြင်း)။

၃။ ဘာကြောင့် .gitignore လိုအပ်သလဲ?

React ပရောဂျက်တွေမှာ node_modules လိုမျိုး အရမ်းလေးလံတဲ့ folder တွေ ပါဝင်ပါတယ်။ ဒါတွေကို GitHub ပေါ်တင်ဖို့ မလိုအပ်ပါဘူး (တင်ရင်လည်း အရမ်းကြာပါလိမ့်မယ်)။ .gitignore ဆိုတဲ့ ဖိုင်လေးထဲမှာ အဲ့ဒီ folder နာမည်ကို ထည့်ထားလိုက်ရင် Git က အဲ့ဒါကို လျစ်လျူရှုသွားမှာ ဖြစ်ပါတယ်။ ဒါဟာ Professional Developer တိုင်း သိထားရမယ့် အချက်ဖြစ်ပါတယ်။

📺 ဗွီဒီယိုဖြင့် လေ့လာရန်

Git & GitHub ဆိုတာဘာလဲ? (မြန်မာလို)

ဒီဗွီဒီယိုမှာ Git ရဲ့ အခြေခံ သဘောတရားတွေကို အလွယ်ကူဆုံး ရှင်းပြထားပါတယ်။ တစ်ခါတည်း ကြည့်ရှုလေ့လာနိုင်ပါတယ်။

Git & GitHub Crash Course (English)

Global Standard အတိုင်း အသေးစိတ် သိရှိလိုသူများအတွက် ပြည့်စုံသော သင်ခန်းစာ ဖြစ်ပါတယ်။

အကြံပြုချက်

Git ကို မသုံးခင် သင့်စက်ထဲမှာ Git ရှိမရှိ git --version နဲ့ အရင်စစ်ပါ။ မရှိရင် git-scm.com မှာ install အရင်လုပ်ပါ။

🏋️ လက်တွေ့လေ့ကျင့်ခန်း: Your First Repo

GitHub မှာ အကောင့်တစ်ခုဖွင့်ပါ။ ပြီးရင် "my-first-repo" ဆိုပြီး Repository တစ်ခုဆောက်ပါ။ အပေါ်က commands တွေကို သုံးပြီး သင့်စက်ထဲက ဖိုင်တစ်ခုကို အဲ့ဒီ repo ဆီ ရောက်အောင် ပို့ကြည့်ပါ။
