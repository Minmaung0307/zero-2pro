SSH Key: GitHub အတွက် မှော်သော့တစ်ချောင်း 🔑

၁။ SSH ဆိုတာ ဘာလဲ? ဘာလို့ သုံးရတာလဲ?

"မင်း GitHub ပေါ် ကုဒ်တွေ တင်လိုက်တိုင်း Password ဒါမှမဟုတ် Token တွေ ခဏခဏ ရိုက်နေရလို့ စိတ်ညစ်နေပြီလား?"

SSH (Secure Shell) ဆိုတာ သင့်ကွန်ပျူတာနဲ့ GitHub ကြားမှာ တည်ဆောက်လိုက်တဲ့ "လျှို့ဝှက်ဥမင်လမ်း" တစ်ခုလိုပါပဲ။ သူက သော့နှစ်ချောင်း သုံးပါတယ်။

Public Key (သော့ခလောက်)

ဒါကို GitHub ပေါ်မှာ သွားတင်ထားရပါမယ်။ လူတိုင်း မြင်လို့ရပါတယ်။

Private Key (သော့အစစ်)

ဒါက သင့်စက်ထဲမှာပဲ ရှိနေရပါမယ်။ ဘယ်သူ့ကိုမှ ပေးမသိရပါဘူး။

ဘာကြောင့်သုံးရလဲ: Password ခဏခဏ ရိုက်စရာမလိုတော့တဲ့အပြင်၊ Hacker တွေက သင့် Password ကို သိသွားရင်တောင် သင့်စက်ထဲက "Private Key" မရှိရင် GitHub ထဲ ဝင်လို့မရတဲ့အတွက် အရမ်းလုံခြုံပါတယ်။

၂။ SSH Key ပြုလုပ်ပုံ အဆင့်ဆင့်

အဆင့် (က) သော့အသစ် ထုတ်လုပ်ခြင်း

Terminal ကိုဖွင့်ပြီး အောက်ပါကုဒ်ကို ရိုက်ထည့်ပါ။ (Email နေရာမှာ သင့် GitHub email ကို အစားထိုးပါ)

ssh-keygen -t ed25519 -C "your_email@example.com"

မှတ်ချက်: Enter ၃ ခါ ဆက်တိုက်နှိပ်ပေးလိုက်ပါ။

အဆင့် (ခ) သော့ကို ကော်ပီကူးခြင်း (Public Key)

ထုတ်လိုက်တဲ့သော့ထဲက "Public Key" ကို GitHub ကို ပေးဖို့အတွက် Copy ကူးရပါမယ်။

# Windows အတွက် (CMD/PowerShell)

clip < ~/.ssh/id_ed25519.pub

# Mac အတွက်

pbcopy < ~/.ssh/id_ed25519.pub

အဆင့် (ဂ) GitHub ပေါ်သို့ တင်ခြင်း

  1. GitHub.com သို့သွားပြီး Login ဝင်ပါ။
  2. ညာဘက်အပေါ်က သင့်ပုံလေးကိုနှိပ်ပြီး Settings ကိုသွားပါ။
  3. ဘယ်ဘက် Menu ထဲက SSH and GPG keys ကိုနှိပ်ပါ။
  4. New SSH Key ဆိုတဲ့ ခလုတ်အစိမ်းလေးကို နှိပ်ပါ။
  5. Title မှာ "My PC" လို့ပေးပြီး၊ Key နေရာမှာ ခုနက Copy ကူးလာတာကို Paste လုပ်လိုက်ပါ။

၃။ အလုပ်လုပ်မလုပ် စမ်းသပ်ခြင်း

အကုန်ပြီးသွားရင် Terminal မှာ ဒါလေးရိုက်ပြီး စစ်ကြည့်ပါ။

ssh -T git@github.com

"Hi [YourName]! You've successfully authenticated..." ဆိုတဲ့ စာတန်းလေး ပေါ်လာရင်တော့ သင်အောင်မြင်သွားပါပြီ။ အခုကစပြီး GitHub သုံးရင် Password ရိုက်စရာ မလိုတော့ပါဘူး။

How to Setup SSH Keys for GitHub

အဖြစ်များသော အမှားများ (Troubleshooting)

"Permission denied (publickey)"

အဖြေ: GitHub ပေါ်မှာ SSH Key တင်ဖို့ ကျန်နေတာ သို့မဟုတ် Copy ကူးတာ မှားနေလို့ပါ။ Key အသစ်ပြန်ထုတ်ပြီး ပြန်တင်ကြည့်ပါ။

"Could not resolve hostname"

အဖြေ: အင်တာနက် လိုင်းမကောင်းတာ သို့မဟုတ် `git@github.com` ဆိုတဲ့ စာလုံးပေါင်း မှားနေတာပါ။
