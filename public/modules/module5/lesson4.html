🛠 Module 5: Mind Reader Arena

"Context API နဲ့ Global State Management ရဲ့ ရှုပ်ထွေးတဲ့ အမှားတွေကို ရှာဖွေပြီး App ရဲ့ ဦးနှောက်ကို စနစ်တကျ ထိန်းချုပ်ပါ။"

Challenge 1: The Missing Provider Component

"ကျောင်းသားက Context ကို သုံးချင်လို့ Component တွေကို ပတ်ပေးထားပေမယ့် Data တွေက Dashboard ဆီ မရောက်ပါဘူး။"

BUGGY CODE // App.jsx
<UserContext> // ❌
  <Dashboard />
</UserContext>
အဖြေမှန်ကို ကြည့်မည်
Context Object ကို တိုက်ရိုက် component အဖြစ် သုံးလို့မရပါဘူး။ Data ဖြန့်ဝေပေးမယ့် .Provider component ကို သုံးရပါမယ်။
✅ ပြင်ဆင်ရန်: <UserContext.Provider> ... </UserContext.Provider>

Challenge 2: The Value Prop Error

"Provider ကို သုံးထားပေမယ့် Child component ထဲမှာ useContext နဲ့ ခေါ်လိုက်ရင် undefined ပဲ ထွက်နေပါတယ်။"

BUGGY CODE <ThemeContext.Provider> // ❌
  <App />
</ThemeContext.Provider>
အဖြေမှန်ကို ကြည့်မည်
Provider မှာ ဘယ် data တွေကို ဖြန့်ဝေပေးမလဲဆိုတဲ့ value prop မပါလို့ပါ။ အဲ့ဒီ value ထဲမှာမှ state တွေ သို့မဟုတ် data တွေကို ထည့်ပေးရပါတယ်။
✅ ပြင်ဆင်ရန်: <ThemeContext.Provider value={theme}>

Challenge 3: Violating Hook Rules

"User ရှိမှ data ကို ယူချင်တာပါ။ ဒါပေမယ့် React က 'Hooks can only be called at the top level' ဆိုတဲ့ Error ပြနေပါတယ်။"

BUGGY CODE if (user) {
  const data = useContext(UserContext); // ❌
}
အဖြေမှန်ကို ကြည့်မည်
React Hooks (useContext အပါအဝင်) တွေကို if-statement တွေ၊ loop တွေ၊ function ငယ်တွေထဲမှာ ရေးလို့မရပါဘူး။ Component ရဲ့ အပေါ်ဆုံး (Top level) မှာပဲ ရေးရပါမယ်။
✅ ပြင်ဆင်ရန်: useContext ကို အပေါ်ဆုံးမှာ ခေါ်ပြီးမှ ရလာတဲ့ data ကို if ထဲမှာ သုံးရပါမယ်။

Challenge 4: Import Mismatch

"Context ကို တခြားဖိုင်ကနေ လှမ်းယူပေမယ့် 'UserContext is not defined' လို့ Error တက်နေပါတယ်။"

BUGGY CODE // ContextFile.js: export const UserContext = createContext();

// Navbar.js:
import UserContext from './UserContext'; // ❌
အဖြေမှန်ကို ကြည့်မည်
Context ကို **Named Export** (export const ...) အနေနဲ့ ထုတ်ထားရင် import လုပ်တဲ့အခါ တွန့်ကွင်း { } နဲ့ ယူရပါမယ်။
✅ ပြင်ဆင်ရန်: import { UserContext } from './UserContext';

Challenge 5: Out of Context Range

"Header ထဲမှာ Context data ကို သုံးထားပါတယ်။ ဒါပေမယ့် Header ဟာ Provider ရဲ့ အပြင်ဘက်မှာ ရှိနေရင် ဘာဖြစ်မလဲ?"

BUGGY CODE <Header /> // ❌ Context used here
<UserContext.Provider value={data}>
  <Dashboard />
</UserContext.Provider>
အဖြေမှန်ကို ကြည့်မည်
useContext ကို သုံးမယ့် component ဟာ သက်ဆိုင်ရာ Provider ရဲ့ အထဲမှာ (as a child) ရှိနေရပါမယ်။ အပြင်ဘက်ကနေ လှမ်းသုံးရင် context data တွေကို ရမှာမဟုတ်ပါဘူး။
✅ ပြင်ဆင်ရန်: Header ကို Provider ရဲ့ အထဲကို ရွှေ့ပေးရပါမယ်။

Brain Error (၅) ခုလုံးကို ပြင်နိုင်ပြီလား?
