Components: Website ရဲ့ အစိတ်အပိုင်းများ

React မှာ Website တစ်ခုလုံးကို Function လေးတွေ နဲ့ တည်ဆောက်ပါတယ်။ ဥပမာ - Facebook မှာဆိုရင် "Post" တစ်ခုဟာ Component တစ်ခုပါ။ "Comment Box" ဟာ နောက်ထပ် Component တစ်ခုပါ။

Component တစ်ခုရဲ့ ပုံစံ:

function Welcome() {
  return <h1>မင်္ဂလာပါ React ကမ္ဘာမှ ကြိုဆိုပါတယ်။</h1>;
}

မှတ်ချက်: Component နာမည်ကို အမြဲတမ်း အက္ခရာအကြီး (Capital Letter) နဲ့ စရပါမယ်။

JSX ဆိုတာ ဘာလဲ?

JSX ဆိုတာ JavaScript ထဲမှာ HTML ရေးတဲ့ ပုံစံပါ။ HTML နဲ့ အရမ်းတူပေမယ့် အချို့အချက်တွေ ကွဲပြားပါတယ်။

JSX ရဲ့ စည်းမျဉ်းများ:

  • `class` လို့ မသုံးရဘဲ `className` လို့ သုံးရပါတယ်။
  • Tag အားလုံးဟာ အပိတ်ပါရပါမယ် (ဥပမာ- ``)။
  • Return ပြန်တဲ့အခါ Element တစ်ခုတည်း (Parent တစ်ခုတည်း) ထဲမှာပဲ အကုန်ထည့်ရပါတယ်။

၁.၂ - JSX နှင့် Virtual DOM ၏ နက်နဲသော အလုပ်လုပ်ပုံ

၁။ JSX ဆိုတာ ဘာလဲ? (တကယ်တော့ သူက JavaScript ပါ)

JSX ဆိုတာ JavaScript XML ရဲ့ အတိုကောက်ပါ။ အပေါ်ယံကြည့်ရင် HTML နဲ့ တူပေမယ့် တကယ်တမ်းမှာတော့ သူဟာ အစွမ်းထက်တဲ့ JavaScript object တွေသာ ဖြစ်ပါတယ်။ Browser က JSX ကို တိုက်ရိုက်နားမလည်ပါဘူး။ ကျွန်တော်တို့ ရေးလိုက်တဲ့ <h1>Hello</h1> ကို React က နောက်ကွယ်မှာ React.createElement('h1', null, 'Hello') ဆိုတဲ့ JavaScript code အဖြစ် ပြောင်းလဲပစ်ပါတယ်။

HTML နှင့် ကွဲပြားချက်များ:

  • class အစား className သုံးရခြင်း
  • onclick အစား onClick (camelCase) သုံးရခြင်း
  • Style တွေကို string အစား Object အနေနဲ့ ပေးရခြင်း
  • Tag အားလုံးကို အပိတ် (Self-closing) ထည့်ရခြင်း

ဘာကြောင့်သုံးတာလဲ?

Logic (JavaScript) နဲ့ UI (HTML) ကို တစ်နေရာတည်းမှာ ရောရေးနိုင်လို့ Code တွေကို ပိုမို စနစ်တကျ ရှိစေပါတယ်။ တွန့်ကွင်း { } ကို သုံးပြီး JavaScript variable တွေကို UI ထဲ တိုက်ရိုက် ထည့်သုံးနိုင်ပါတယ်။

၂။ Virtual DOM ဆိုတာ မှော်အတတ်လား?

Browser ရဲ့ DOM (Document Object Model) ကို တိုက်ရိုက်ပြင်တာဟာ အရမ်းကို ဝန်ပိစေပါတယ်။ ဥပမာ - Facebook notification တစ်ခုတက်တိုင်း Browser တစ်ခုလုံးကို ပြန်ဆွဲနေရင် အရမ်းနှေးသွားမှာပါ။ React က ဒါကို Virtual DOM နဲ့ ဖြေရှင်းပါတယ်။

React က လက်ရှိ UI ရဲ့ "ပုံတူပွား (Copy)" တစ်ခုကို memory ထဲမှာ သိမ်းထားပါတယ်။ တစ်ခုခု ပြောင်းလဲသွားရင် တကယ့် Browser DOM ကို မပြင်ခင်မှာ အဲ့ဒီ Copy (Virtual DOM) ကို အရင်ပြင်ပါတယ်။ ပြီးမှ အဟောင်းနဲ့ အသစ်ကို နှိုင်းယှဉ်ပြီး (Diffing Algorithm) တကယ်ပြောင်းလဲသွားတဲ့ "အစိတ်အပိုင်းလေး" ကိုပဲ Browser DOM မှာ သွားပြင်ပါတယ်။ ဒါကြောင့် React App တွေဟာ အရမ်းမြန်နေရတာ ဖြစ်ပါတယ်။

၃။ Fragments: မလိုအပ်တဲ့ Div တွေကို ဖယ်ရှားခြင်း

JSX မှာ return ပြန်တဲ့အခါ Element အားလုံးကို တစ်ခုတည်းသော Parent အောက်မှာပဲ ထားရပါတယ်။ အရင်ကတော့ <div> တွေနဲ့ ပတ်ခဲ့ကြပါတယ်။ ဒါပေမယ့် မလိုအပ်ဘဲ div တွေ များလာတာဟာ UI structure ကို ရှုပ်ထွေးစေပါတယ်။ React က ဒါကို <React.Fragment> သို့မဟုတ် အတိုကောက် <> ... </> နဲ့ ဖြေရှင်းပေးပါတယ်။ သူက UI မှာ ဘာ tag မှ ထပ်မတိုးစေဘဲ JSX စည်းမျဉ်းကို ပြည့်စုံစေပါတယ်။
