User ရိုက်သမျှကို လက်ခံခြင်း (Inputs) ⌨️

Website တစ်ခုမှာ User က စာရိုက်တာ၊ ခလုတ်နှိပ်တာတွေကို React က ဘယ်လို သိသလဲ? အဲ့ဒါကို Event Handling လို့ ခေါ်ပါတယ်။

onClick နှင့် onChange:

  • onClick: ခလုတ်နှိပ်တဲ့အခါ အလုပ်လုပ်စေချင်ရင် သုံးပါတယ်။
    <button onClick={() => alert('Hello')}>Click Me</button>
  • onChange: Input box ထဲမှာ စာရိုက်တဲ့အခါ အလုပ်လုပ်စေချင်ရင် သုံးပါတယ်။
    <input onChange={(e) => setName(e.target.value)} />

Pro Tip: Controlled Components

React မှာ Input ထဲကစာကို state ထဲ သိမ်းလိုက်တာကို Controlled Component လို့ ခေါ်ပါတယ်။ ဒါက အကောင်းဆုံး နည်းလမ်းပါပဲ။

လက်တွေ့ Exercise: Enrollment Form

ကျောင်းသားနာမည်ကို ရိုက်ထည့်ရင် ဘေးနားမှာ "Welcome, [Name]" ဆိုပြီး ချက်ချင်းပေါ်လာမယ့် UI လေးတစ်ခုကို React မှာ ရေးကြည့်ပါ။

Controlled vs Uncontrolled Inputs

Form တွေကို စနစ်တကျ ထိန်းချုပ်နည်း

၂.၂ - Event Handling နှင့် User ဆီမှ Data လက်ခံခြင်း

၁။ React တွင် Event ကိုင်တွယ်ပုံ

HTML မှာ onclick လို့ သုံးခဲ့ပေမယ့် React မှာတော့ Camel Case ဖြစ်တဲ့ onClick ကို သုံးရပါတယ်။ အဲ့ဒီမှာ String ပေးမယ့်အစား Function တစ်ခုကို လှမ်းပေးရပါတယ်။ React ဟာ Synthetic Events လို့ခေါ်တဲ့ စနစ်ကို သုံးထားလို့ Browser တိုင်း (Chrome, Safari, Firefox) မှာ အလုပ်လုပ်ပုံ တစ်သမတ်တည်း ဖြစ်နေစေပါတယ်။

၂။ Controlled Components (Form များကို ထိန်းချုပ်ခြင်း)

Web App တစ်ခုမှာ Input box တွေထဲက စာသားတွေကို ဘယ်လိုယူမလဲ? React ရဲ့ အကောင်းဆုံးနည်းလမ်းကတော့ Controlled Component ပါ။ ဆိုလိုတာက Input ထဲက စာသားဟာ React State ပေါ်မှာ ၁၀၀ ရာခိုင်နှုန်း မူတည်နေတာပါ။

အဆင့် (၃) ဆင့်:

  1. စာသားသိမ်းဖို့ State တစ်ခုဆောက်ပါ။
  2. Input ရဲ့ value ကို အဲ့ဒီ state နဲ့ ချိတ်ပါ။
  3. onChange event ထဲမှာ User ရိုက်လိုက်တဲ့စာကို e.target.value ကနေယူပြီး state ကို update လုပ်ပေးပါ။

ဒီနည်းလမ်းက အရမ်းအစွမ်းထက်ပါတယ်။ User က စာရိုက်နေတုန်းမှာတင် စာလုံးရေ ဘယ်နှလုံးရှိပြီလဲ စစ်တာ၊ အမှားပါရင် ချက်ချင်း အနီရောင်ပြတာ (Validation) တွေကို လွယ်လွယ်ကူကူ လုပ်နိုင်ပါတယ်။
