၁။ Prop Drilling ဆိုတာ ဘာလဲ?

ဥပမာ - သင့်မှာ Component (A) ရှိတယ်၊ အဲ့ဒီအထဲမှာ (B) ရှိတယ်၊ (B) ထဲမှာ (C) ရှိတယ် ဆိုပါစို့။ (A) ဆီက Data ကို (C) ဆီ ပို့ချင်ရင် (B) ကို ကြားခံအနေနဲ့ အလကားသက်သက် ဖြတ်ခိုင်းရတာကို Prop Drilling လို့ ခေါ်ပါတယ်။ ဒါဟာ App ကြီးလာရင် အရမ်းကို ထိန်းသိမ်းရ ခက်ခဲစေပါတယ်။

၂။ Context API: ဗဟိုသိုလှောင်ရုံ

React Context ဟာ App တစ်ခုလုံးရဲ့ "Cloud Storage" လိုပါပဲ။ Data တွေကို အပေါ်ဆုံးမှာ သိမ်းထားပြီး လိုချင်တဲ့ Component က လှမ်းဆွဲယူလိုက်ရုံပါပဲ။

၅.၁ - Context API: Global မှတ်ဉာဏ် တည်ဆောက်ခြင်း

၁။ Prop Drilling ပြဿနာ

React မှာ ဒေတာတွေကို Parent ကနေ Child ဆီ ပို့ရပါတယ်။ ဒါပေမဲ့ Component တွေက အထပ်ထပ် ဖြစ်လာတဲ့အခါ (ဥပမာ- App -> Page -> Sidebar -> UserCard) အပေါ်ဆုံးက User data ကို အောက်ဆုံးဆီ ပို့ဖို့ ကြားထဲက Component တွေက အလကားသက်သက် လက်ဆင့်ကမ်းပေးနေရပါတယ်။ ဒါကို Prop Drilling လို့ ခေါ်ပါတယ်။ ဒါဟာ ကုဒ်တွေကို ရှုပ်ထွေးစေပြီး ထိန်းသိမ်းရ ခက်စေပါတယ်။

Context API ဟာ ဒီပြဿနာကို ရှင်းပေးတဲ့ "Cloud Storage" လိုပါပဲ။ ဒေတာတွေကို တစ်နေရာတည်းမှာ သိမ်းထားပြီး လိုချင်တဲ့ Component က ဘယ်နေရာကနေမဆို လှမ်းယူသုံးနိုင်အောင် လုပ်ပေးပါတယ်။

၂။ Provider နှင့် Consumer သဘောတရား

Context မှာ အဓိက အစိတ်အပိုင်း (၂) ခု ရှိပါတယ်။ Provider ဆိုတာကတော့ ဒေတာတွေကို ဖြန့်ဝေပေးတဲ့ Router စက်ကြီးလိုပါပဲ။ သူ့ရဲ့ အောက်မှာရှိသမျှ Component အားလုံးဟာ ဒေတာတွေကို ရရှိနိုင်ပါတယ်။ useContext Hook ကတော့ Wi-Fi ဖမ်းတဲ့ ဖုန်းလိုပါပဲ။ Provider ဆီက ဒေတာတွေကို လှမ်းဖမ်းယူပြီး အသုံးပြုခွင့် ပေးပါတယ်။
